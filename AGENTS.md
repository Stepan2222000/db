# Правила для агентов

## Принципы кода

- Не делать глупых фолбеков, не совершать действия нацеленные на overengineering
- Не проверять то, что Python/OS проверят сами — если файл не читается, ошибка от OS понятнее нашей. Код проверяет только бизнес-логику
- Не хардкодить значения, которые оператор может менять
- Использовать `logging` вместо `print`
- Тестировать всё в терминале

## Спецификация

Перед реализацией читай:
- `docs/SPEC.md` — полная спецификация
- `docs/SPEC.flow.yaml` — flow-граф с техническими деталями
