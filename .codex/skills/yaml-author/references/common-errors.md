# Частые Ошибки YAML

Короткий список типовых проблем, которые стоит проверять в первую очередь.

## 1. Двоеточие внутри некавычённого значения

Плохо:

```yaml
condition: сообщение при `mode: sleeping`
```

Правильно:

```yaml
condition: "сообщение при `mode: sleeping`"
```

Почему: `:` внутри обычной строки может быть воспринят как начало вложенного `key: value`.

Частая ошибка парсера:

```text
Nested mappings are not allowed in compact mappings
```

## 2. Смешивание табов и пробелов

YAML ожидает отступы пробелами. Таб в отступе часто ломает парсинг.

## 3. Неровные отступы в списках

Плохо:

```yaml
links:
  - to: next-step
     condition: ok
```

Правильно:

```yaml
links:
  - to: next-step
    condition: ok
```

## 4. Markdown или текст с `:` лучше брать в кавычки

Это особенно важно для полей вроде:

- `condition`
- `title`
- `message`
- коротких inline-описаний

## 5. Многострочный текст лучше уносить в `|`

Для длинного markdown-текста безопаснее использовать block scalar:

```yaml
body_markdown: |
  Первая строка
  Вторая строка
```

## 6. Не использовать markdown-таблицы в `body_markdown`

Плохо:

```yaml
body_markdown: |
  | Поле | Значение |
  |------|----------|
  | `chatId` | обязателен |
```

Лучше:

```yaml
body_markdown: |
  - `chatId` — обязателен
  - `timezone` — таймзона для записей
```

Почему: в текущем UI таблицы внутри карточек рендерятся плохо и ломают читаемость. Для таких блоков использовать списки.
